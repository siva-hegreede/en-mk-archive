# en-mk-archive

## 项目简介

`en-mk-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库旨在提供一种结构化的方式，将不同来源或不同用途的静态页面集中存放，便于长期保存、版本管理以及通过 GitHub Pages 或其他静态托管服务进行访问。

仓库本身不针对任何特定域名或网站，仅作为通用归档容器。

## 目录说明

```
en-mk-archive/
├── pages/          # 存放所有独立的 HTML 页面文件
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
├── index.html      # 可选的入口页面，用于展示归档页面列表
└── README.md       # 本文件
```

- **pages/**：每个独立 HTML 文件直接置于此目录下，或按日期/主题创建子目录进行组织。
- **assets/**：如果多个页面共享样式或脚本，可在此处存放，页面通过相对路径引用。
- **index.html**：建议编写一个简单的导航页面，罗列归档页面的标题与链接，方便浏览。

## 页面归档说明

- 本仓库仅存放静态 HTML 文件及其相关资源，不包含后端逻辑或动态内容。
- 归档页面可能来自个人项目、学习笔记、临时展示页或第三方授权转载（请确保版权合规）。
- 每个页面应保持独立，避免依赖外部在线资源（除非必要），以确保离线或长期可用性。
- 如需更新已有归档页面，请保持文件名不变，并在提交信息中注明修改内容。

## 维护说明

- 本仓库由维护者不定期更新，欢迎通过 Issue 或 Pull Request 提出建议或提交新的归档页面。
- 提交新页面时，请确保：
  - HTML 文件放置在 `pages/` 目录下。
  - 文件命名简洁、语义化，避免特殊字符。
  - 如引用 `assets/` 中的资源，请使用相对路径。
- 仓库默认使用 `main` 分支，GitHub Pages 可配置为从 `main` 分支的根目录发布，也可根据需要调整。
